** Executive summary
This report documents the end-to-end outcome of Task 2d:
- bitstream generation from the open-source flow,
- manual programming on FPGA hardware,
- bring-up/self-test validation against the fixed Task-2 vector,
- evidence packaging for review.

Current status reported by implementer:
- bitstream loaded on board flash,
- self-test working on hardware.

** Scope
- Fixed vector validation for the Task-2 matmul kernel.
- Bring-up performed with manual programming commands (`openFPGALoader`).
- Focus is functional bring-up evidence, not performance benchmarking.

** Hardware and software setup
- Board: YPCB-00338-1P1 (Kintex-7 class target)
- Cable/programmer tool: `openFPGALoader`
- Build system: Nix flake in this repository
- Key derivations:
  - `.#matmul-bitstream`
  - `.#matmul-selftest-bitstream`
  - `.#matmul-bitstream-fasm`

** Procedure used
1. Build bitstream artifacts.
2. Scan JTAG and program bitstream manually.
3. Run board self-test (LED PASS/FAIL protocol).
4. Compare expected/got values using `sim/check_hw_result.py` when host-observed output is available.

Representative commands:

#+begin_src bash
nix build .#matmul-selftest-bitstream -L
BIT="$(readlink -f result)"
openFPGALoader --scan-jtag
openFPGALoader "$BIT"
python3 sim/check_hw_result.py --print-expected
#+end_src
