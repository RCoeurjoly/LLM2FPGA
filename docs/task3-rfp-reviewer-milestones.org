* Task 3 reviewer milestones for the RfP

This document defines a reviewer-facing reproduction path for the current Task 3
state. It separates the *pinned baseline deliverables* from the *frontier local
compiler smoke*, because those two checkpoints answer different questions.

** Why this split exists

- The pinned flake already reproduces the current baseline Task 3 deliverables
  3a through 3e.
- The current TorchAO/TinyStories frontier work depends on a local
  =torch-mlir-opt= with post-pin fixes; that is not the same claim as the
  pinned 3a through 3e deliverables.
- Reviewers therefore need one script that records both:
  - the stable flake-backed Task 3 artifacts
  - the current local fast-loop/compiler checkpoint

** One-command reviewer run

#+begin_src bash
nix build .#tiny-stories-1m-utilization -L
scripts/dev/run-task3-rfp-milestones.sh /tmp/task3-rfp-milestones
#+end_src

The script emits a self-contained output directory with:

- =README.md=: milestone summary
- =summary.env=: shell-safe resolved artifact paths, milestone status, and
  deliverable-to-artifact mapping
- =torchao-attention-block-linalg.mlir=: local fast-loop frontier artifact
- =torchao-attention-block-profile.txt=: op profile for that frontier artifact

If a local patched =TORCH_MLIR_OPT= is available, the same run also emits the
full TinyStories TorchAO downstream smoke artifacts through =hs-ext= plus the
captured =hs-ext -> hw0= failure log.

** Milestone definitions

*** Milestone A: local fast-loop frontier checkpoint

Definition:
- The small TorchAO attention-block reproducer reaches local =linalg=.
- The emitted artifact still contains the expected float-heavy families that
  define the current remaining work: dynamic qparam computation, =LayerNorm=,
  and softmax residue.

Default artifact:
- =/tmp/task3-rfp-milestones/torchao-attention-block-linalg.mlir=

Acceptance signal:
- The generated profile reports the expected mixed integer/float shape rather
  than collapsing back to plain float.
- =summary.env= records:
  - =milestone_a_status=pass=
  - =milestone_a_acceptance=mixed-int-float-linalg-profile=

*** Milestone B: pinned baseline Task 3 deliverables

Definition:
- The current flake reproduces the official Task 3 baseline artifacts:
  - 3a snapshot
  - 3b pre-CIRCT MLIR
  - 3c CIRCT-emitted SystemVerilog
  - 3d Yosys RTLIL
  - 3e canonical Yosys-mapped utilization report

Acceptance signal:
- =summary.env= records:
  - =milestone_b_status=pass=
  - =deliverable_3a_*= through =deliverable_3e_*= keys
  - the exact gate command for each deliverable
  - direct paths for reviewer-relevant files such as =config.json=,
    =pytorch_model.bin=, =sv/main.sv=, =sources.f=, =summary.txt=,
    =summary.json=, =stat.json=, and the mapped JSON emitted by
    =.#tiny-stories-1m-utilization-json=

*** Milestone C: local TinyStories TorchAO downstream smoke

Definition:
- With a local patched =torch-mlir-opt=, the TinyStories TorchAO path should
  reach:
  - =linalg=
  - =cf=
  - =handshake=
  - =hs-ext=
- The current expected result is that =hs-ext -> hw0= still fails because the
  artifact still contains floating-point-heavy residue.

Acceptance signal:
- =summary.env= reports:
  - =frontier_status=expected-hw0-failure=
  - =milestone_c_status=pass=
  - =tiny_stories_torchao_hs_ext_profile_text=
  - =frontier_reason= quoting the first legalization failure
- If the local compiler state has advanced, the same keys record that as an
  unexpected improvement instead of silently changing behavior.

** Reviewer notes

- Milestone B is the reproducible baseline claim.
- The canonical Task 3 utilization gate is:

#+begin_src bash
nix build .#tiny-stories-1m-utilization -L
#+end_src

- Milestones A and C are the reproducible compiler-roadmap checkpoints.
- The script prefers local fast iteration where that is the point of the
  milestone, and uses the pinned flake where the point is stable reproduction.
- =summary.env= is intended to be shell-safe:

#+begin_src bash
source /tmp/task3-rfp-milestones/summary.env
printf '%s\n' "$deliverable_3c_main_sv"
#+end_src
