*** Variant 1: Port Netnames Only
Local importer change:
- collect the set of signal indices that appear in module ports
- keep netnames only for those port bits
- skip non-port net attrs alongside them

Purpose:
- emulate the best prior JSON pruning result inside the importer itself
- eliminate the enormous ~main.netnames~ candidate-name table without rewriting
  JSON on disk

Measured result:
- ~Command terminated by signal 9~
- max RSS: ~31,053,968 kB~ (~31.1 GB~)
- wall time: ~2:12.04~

Last importer telemetry before kill:
- ~ctx_nets=13,458,082~
- ~ctx_cells=11,770,079~
- ~ctx_aliases=18,035,701~
- ~flat_nets=11,700,065~
- ~rss_kb=30,827,684~

What improved:
- the huge top-level ~main~ candidate-name storage was eliminated
- ~module_net_slots=0~ and ~module_candidate_slots=0~ for the sampled modules
- the importer reached materially farther into the real graph before dying

What did not improve enough:
- ~ctx_aliases~ was still very large
- max RSS still converged to the same ~31 GB~ ceiling

*** Variant 2: Suppress Secondary Net Aliases
Local importer change on top of Variant 1:
- do not add secondary aliases into ~ctx->net_aliases~
- do not append extra entries into ~net->aliases~
- keep only the primary net name created by ~createNet(...)~

Measured result:
- ~Command terminated by signal 9~
- max RSS: ~31,195,112 kB~ (~31.2 GB~)
- wall time: ~2:05.50~

Last importer telemetry before kill:
- ~ctx_nets=13,804,411~
- ~ctx_cells=12,089,062~
- ~ctx_aliases=13,823,332~
- ~flat_nets=12,000,065~
- ~rss_kb=30,700,976~

What improved:
- alias growth was cut from "far above net count" to essentially
  ~ctx_aliases ~= ctx_nets~
- importer frontier moved farther again

Interpretation:
- alias storage is real overhead
- but it is still not enough on its own to get below the machine's kill point

*** Variant 3: Short Synthetic Internal Names
Local importer change on top of Variant 2:
- for non-top-level imported nets, cells, and constant helpers, stop using long
  hierarchical names
- instead synthesize short names like ~\$nNNN~ and ~\$cNNN~
- top-level naming remains unchanged

Purpose:
- reduce ~ctx->id(...)~ pressure and stored string size after the metadata-heavy
  parts had already been cut down

Measured result:

- max RSS: ~31,314,008 kB~ (~31.3 GB~)

** Current Interpretation
The local iteration changed the picture in an important way.

What is now clearly reduced:
- top-level ~main.netnames~ candidate-name storage
- secondary alias-map growth
- long internal imported names

What remains:
- repeated ~create_or_get_net(...)~ work during leaf-cell import
- growth of the real imported flat graph itself
- eventual convergence to a machine kill around ~31 GB~ RSS

So the current frontier is no longer "fix the first obvious importer bug" or
"trim obvious naming metadata". Those helped, but they are not sufficient.

The remaining problem now looks like:
- high per-net / per-cell importer memory cost during real graph construction
- not primarily candidate-name storage anymore

** Practical Conclusion
These local importer changes are real improvements:
- baseline frontier: about ~11.3M~ imported nets before kill
- best local frontier here: about ~15.0M~ imported nets before kill

But they still do not complete the TinyStories import on this machine.

So this is not yet a complete fix. It is a meaningful reduction of importer
overhead and a clearer diagnosis of what remains.

** Recommended Next Steps
1. Keep Variant 1 and Variant 2 as plausible importer-side cleanup candidates:
   - port-netnames-only
   - no secondary aliases
2. Treat Variant 3 as a useful experiment but not yet obviously production-ready:
   - it improves frontier
   - but it also leads to long high-system-time stalls
3. Focus the next nextpnr-xilinx investigation on actual graph-build footprint:
   - net object size
   - cell object size
   - per-port connectivity storage
   - repeated hierarchy flattening cost
4. If another iteration is needed, add counters for:
   - total port objects created
   - total driver/user links
   - total constant-helper cells
   - memory attributed to ~ctx->nets~, ~ctx->cells~, and port maps
