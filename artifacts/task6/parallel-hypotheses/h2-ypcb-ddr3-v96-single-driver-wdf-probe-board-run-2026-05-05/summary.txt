Interpretation:
v96 fixed the v95 multi-driver build issue. Native WDF push does occur, so the push signal is not entirely wrong. However, the captured native PHY write-data event happens with the FIFO empty while pop/slave/master events are asserted and native selection is active. This points to the pop timing: the FIFO is drained before the observed PHY write-data event, likely because pop is tied to the LiteDRAM slave wrdata event rather than the actual PHY/master wrdata event.

Next surgical action:
For v97, keep the single-driver patch and compact status byte, but pop the FIFO on native master/PHY wrdata event instead of native slave wrdata event, or guard pop so empty cannot be consumed. The goal is to keep accepted native WDF data resident until the PHY write-data event.
