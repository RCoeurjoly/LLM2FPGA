Interpretation:
v97 is a real improvement over v96: native readback is no longer zero. Popping on the master/PHY write-data event keeps the FIFO nonempty at the captured event, and native writes now affect memory. The remaining bug is data correctness: all native read samples return the same wrong constant word, not the per-address expected pattern. This points to wrong beat/phase/chunk selection, FIFO advancement, or DFI data/mask lane mapping, not missing native writes.

Next surgical action:
For v98, preserve master-event pop and expose whether the same FIFO word is being replayed or the wrong DFI beat/lane is selected. Useful compact data: push_count, pop_count, master_event_count, slave_event_count, low data bits, and low write-enable bits at the captured event.
